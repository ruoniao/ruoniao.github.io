    第一部分：OpenFlow 概述

OpenFlow是什么

开源协议

Flow 个人理解泛指网络数据流量

为什么学习OpenFlow

  • 实践SDN的首选
  • 主流的南向接口
  • google P4和 PISA的前身

OpenFlow的起源

斯坦福大学(The McKeown Group)，首先提出OpenFlow，先于SDN概念

  • Ethane项目是OpenFlow的前身

    • 集中式，主动式，基于Flow的控制

  • 2008年的OpenFlow论文

  • OpenFlow最初用于网络实验

    • OpenFlow是为了给研究人员提供一个开放平台，鼓励设备厂商在产品中支持OpenFlow,研究人员就可在一个真实的网络环境中验证新的网络协议，架构和算法的原型。

OpenFlow是什么

SDN != OpenFlow

开放的南向接口协议

数据平面编程的API

通用转发抽象模型

网络X86指令集(博科公司工程师提出)

开发人员可以通过SDN控制器编程，控制器实现基于OpenFlow协议对SDN数据平面设备的转发控制(就是对流量的match和action执行动作)，前提是次数据平面设备支持OpenFlow协议。

传统网络转发模型

  • 传统网络设备各个模块分工明确，各个功能模块堆叠提供统一服务

OpenFlow 通用转发抽象模型

  • 颗粒度上实现控制器对 Flow(数据流)的控制

  • 例如：可以将 Flow Table0配置成Mac地址学习表，Flow Table1配置成Mac转发表
  • 传统的设备的各个模块可以通过流表这种抽象的模型通过编程实现。核心是实现了match,action

第二部分：OpenFlow通用抽象

OpenFlow中的术语

术语 描述
Byte 一个8bit的序列，网络处理的基本单元
Packet 以太帧，包括header和payload
Port 端口，packets进出OpenFlow pipline的地方，由物理端口，Switch定义的logic端口，OpenFlow协议定义的reserved端口组成
Pipline 多个流表组成的流水线。OFS中实现matching,转发和packet修改的flow table 流水线
Flow table 流表。pipline的一个stage,流表，定义的对数据匹配和动作规定的表，包括flow entries
Flow entry 流条目。flow table中的元素，作用是用于packets查找和处理，包括用于包匹配的一组match fields,描述匹配顺序的priority，跟踪记录packets的一组counters,一组待用的instructions。可以简单理解为流表的字段或者属性。就是表的属性
Match Fileld 匹配域。packet匹配查找时使用的域，包括packet header，输入端口和metadata值，一个match field可以是通配符(匹配任意字符),也可以是bitmasked型
Metadata 元数据。一个Maskable register value 用于table之间的信息传递
Instruction 指令。当一个packet 匹配某个flow entry时，对用的instruction 描述特定的OpenFlow处理。一个instruction可以修改pipeline processing过程，比如直接把packets发送给另一个flow table，也可以包含一组actions用来添加到action set,或包含一个action list 立即应用给packet
Action 动作。转发packet到某个端口或修改packet的一个操作，比如减小TTL值。
Action set 动作组。多个action动作组成的集合。
Group 多个动作桶组成的组。一个action buckets 列表，从其中选择一个或多个buckets用在每个packet上
Action bucket 动作桶。一组actions以及相关参数，在group中定义
Tag 标签。通过push和pop actions插入packet或从packet删除一个header
Outermost tag 离packet起始位置最近的一个tag
Crontroller 控制器。通过OpenFlow 协议与OFS交互的设备。
Meter 一个switch基本单位。能测量和控制packets的速率，如果packet速率或byte速率经过mater超出预定义的范围，meter会激活一个meter band ,如果meter band drop packet，就是所谓的 Rate Limiter（速率限制）
OFS Open Flow Switch ；支持Open Flow 协议的交换机

流表项内容

  • 匹配域

    • **

    对进入的每一个数据包定义的一些匹配规则，规则对上了就要对此数据包进行相应的定义的动作

    • 匹配规则：因为协议是在不断升级，所以匹配规则类型也会不断升级，在OF协议1.1之前是OFPMT_STANDARD 类型，1.1版本后的0主流类型是OFPMT_OXM 类型

编写高质量python代码的59个建议之函数

编写高质量python代码的59个建议之函数篇

尽量用异常来表示特殊情况而不是使用None作为返回值

  • 为什么:假如用None作为返回值表示函数的特殊情况，很容易使调用者犯错，因为False,0,已经空字符串，空列表都会被判定为False
  • 怎么做:当函数遇到特殊情况时应该尽量抛出异常，而不是返回None,这样做可以使得调用者看到描述的问题并写相应的逻辑来处理他们。

编程语言的分类

静态语言和动态语言

  • 静态语言：数据类型在编译期间检查，即在写程序时要声明所有变量的数据类型；如Java,c#,c/c++
  • 动态语言：在运行期间才去做数据类型的检查，即在编程中不需给变量声明数据类型，该语言会在第一次赋值给变量时记录下来；如：python,Ruby
